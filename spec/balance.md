## 1. 全局基准公式（建议）

### 1.1 伤害计算基准（简化版）

单次技能基础伤害：

$$
\text{RawDamage} = (ATK_{\text{char}} + ATK_{\text{equip}}) \times \text{Skill倍率} \times (1 + \text{元素加成}) \times (1 - \text{防御系数})
$$

**建议参数**：

- **Skill倍率**：
  - 普攻：1.0–1.2
  - 普通技能：1.2–1.8
  - 大招：2.0–3.0
- **防御简化成一个系数**（不用太复杂一开始）：
  - 防御系数 = DEF_target / (DEF_target + K)，K 取同阶段角色平均 ATK 的 3–5 倍，初期比方说 K=300。
- **元素加成**后面用 ElementPower 表。

---

### 1.2 价格变化基准（每日结算）

某类物品第 t 日价格：

$$
P_{t+1} = P_t \times (1 + \Delta_\text{基础} + \Delta_\text{供需} + \Delta_\text{事件} + \epsilon)
$$

**推荐范围**：

- **Δ基础**：基础微涨微跌，[-0.01, 0.01]
- **Δ供需**：根据玩家/世界交易量，[-0.05, 0.05]
- **Δ事件**：世界事件/宗门决策等，[-0.4, 0.6]（大事才用）
- **ε**：随机噪音，[-0.02, 0.02]

## 2. 装备数值表

### 2.1 品质 vs 词条 & 系数

| 品质 | 稀有度 | 词条数量（前+后缀） | 基础属性系数（相对 N） | 初始 ElementPower 区间 |
|------|--------|---------------------|------------------------|-------------------------|
| 凡品（N） | 白/绿 | 0–1 | 1.0x | 0–5 |
| 灵品（R） | 蓝 | 1–2 | 1.2x | 5–10 |
| 宝品（SR） | 紫 | 2–3 | 1.5x | 10–20 |
| 地品（SSR） | 橙 | 3–4 + 1 特殊 | 1.8x | 20–30 |
| 仙品（SSS） | 金 | 4–5 + 1 唯一 | 2.2x | 30–40 |

基础属性系数用来乘模板的 base_stats，比如同一个模板，一把凡品剑 atk=50，SSS 版就 atk≈110。

| 品阶 | 推荐阶段（境界） | 强化上限 Lv | 每级强化基础属性提升 | 每级强化费用系数（乘基础费用） |

### 2.3 槽位数值预算（同阶段）

以一个"同境界标准角色"为例，推荐装备贡献占总战力的比例：

| 槽位 | 典型主属性 | 当阶段总属性中由装备贡献的比例（大致） |
|------|------------|----------------------------------------|
| 武器 | ATK / 元素 | 35–45% |
| 护具（身） | DEF / HP | 25–30% |
| 饰品 1 | CRIT / 元素 | 10–15% |
| 饰品 2 | 命中 / SPD | 10–15% |
| 法宝 | 特殊效果 | 直接带被动/主动，不强调面板数值 |

**举个中期例子（筑基）**：

- 角色裸 ATK ≈ 80–100
- 一把主力武器 ATK 总加成 ≈ +70–120（基础+强化+词条）
- 其他装备再加一些 ATK 辅助 → 最终总 ATK ≈ 180–250

## 3. 元素数值表

### 3.1 元素强度 ElementPower → 伤害加成

定义一个单元素伤害加成公式：

$$
\text{元素加成}_{elem} = 1 + \frac{\text{ElementPower}_{elem}}{100}
$$

- 建议来自装备的 ElementPower 总和上限：40–60 左右
- 来自技能树/天赋的加成另外算，比如 +20–40

**参考区间（单一元素总和）**：

| 元素强度 EP | 对应伤害系数（仅元素部分） |
|-------------|---------------------------|
| 0 | 1.00x |
| 10 | 1.10x |
| 20 | 1.20x |
| 30 | 1.30x |
| 50 | 1.50x |
| 80（后期） | 1.80x |

可以给一个软上限，如：元素加成来自 EP 最多算到 80（1.8x），再上去就靠特殊效果而不是继续线性。

### 3.2 各元素默认异常状态建议（中期）

配合你之前设定的大概这样：

| 元素 | 基础效果 | 建议数值（中期） |
|------|----------|------------------|
| 火 | 焚灼 DOT | 每回合扣 目标 max_hp × 2%，持续 3 回合，可叠 3 层 |
| 水 | 寒意 / 减速 | 每层 SPD -3%，最多 5 层；3 层以上技能额外 25% 概率小控制 |
| 木 | 毒 / 回复 / 寄生 | 木系 Dot：max_hp × 1.5% 持续 4 回合；治疗技能额外 +10–15% |
| 金 | 破甲 / 反伤 / 护盾 | 破甲层数：每层 DEF -5%，叠 3 层；护盾一般 max_hp × 10–15% |
| 土 | 降暴击 / 减攻 / 定身/嘲讽 | 每层【镇压】：敌人最终伤害 -3%，可叠 3 层；抗暴 +10–20% |

**和元素技能树/装备词条挂钩示例**：

- **装备词条【·焚炎】**：
  - 火元素技能额外附带 1 层焚灼；
  - 焚灼伤害 +30%（变成 2.6%~3% max_hp/回合）
- **装备词条【·寒泉】**：
  - 水系技能施加的寒意多 1 层上限（5→6）；
  - 寒意达 3 层时减速从 30% 提升到 40%。

## 4. 交易数值表

### 4.1 物品类别基础波动

给每类物资一个基础日波动范围 & 典型利润空间：

| 类别 | 基础日波动（不含事件） | 典型买卖利润空间（正常状态） | 说明 |
|------|----------------------|------------------------------|------|
| 药材 / 灵草 | 3–6% | 15–25% | 与炼丹大会强相关 |
| 矿石 / 金属 | 2–5% | 10–20% | 与矿脉事件相关 |
| 兽材 / 妖丹 | 4–7% | 20–30% | 受万兽潮影响大 |
| 丹药成品 | 5–8% | 25–40% | 消耗品，需求大 |
| 装备 / 法器 | 1–4% | 20–50%（视稀有度而定） | 高单价，低频交易 |
| 灵兽相关物品 | 4–8% | 30–50% | 稀有度高，波动大 |

"典型买卖利润空间"指合理低买高卖一轮的目标区间，而不是一天。

### 4.2 世界事件对价格的倍率建议

你前面已经有事件设定，这里给一个统一调参表，做乘法系数：

| 事件 | 影响类别 | 建议价格系数（事件高峰期） |
|------|----------|---------------------------|
| 炼丹大会 | 药材 | ×1.25–1.4 |
| 炼丹大会 | 丹药成品 | ×1.3–1.5 |
| 矿脉枯竭 | 矿石 / 金属装备 | ×1.3–1.6 |
| 万兽潮爆发 | 兽材 / 妖丹 | ×1.3–1.5（短期） |
| 万兽潮平息后可能枯竭期 | 兽材 / 妖丹 | ×1.5–2.0（中长期） |
| 宗门大比 | 战斗丹药 / 装备 | ×1.2–1.3 |
| 灵田丰收（大雨/灵田事件） | 药材 | ×0.7–0.9 |

你可以在每日结算里：

```
today_price = base_price * (1 + daily_noise) * ∏(所有相关事件系数)
```

#### 天赋数值建议

| 天赋 | 效果描述 | 数值建议 |
|------|----------|----------|
| 天机商修 | 所有卖出价格 ×1.15 | +15% 卖价 |
| | 市场预测误差减少一档 | 例如从 ±15% → ±8% |

### 4.4 一个简单"单次交易收益"公式参考

**假设**：

- P_buy：买入价
- P_sell：卖出价
- b_talent：天赋卖价加成（如 0.15）
- b_equip：装备卖价加成（如 0.08）

**则单笔交易利润**：

$$
\text{Profit} = P_\text{sell} \times (1 + b_\text{talent} + b_\text{equip}) - P_\text{buy}
$$

你可以在 UI 里显示"理论利润区间"给玩家一点决策感。
